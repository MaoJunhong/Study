Part1. 引入

一个会说三种语言的人称为trilingual。

一个会说两种语言的人称为bilingual。

那么，一个只会说一种语言的人称做啥？

<安静光环>

American。

这是国际化社区的一个笑话。但是如今，只提供英文版的软件产品已经不能适用。让产品进入国际化市场具有经济意义。

本文将介绍如何翻译RCP插件，让其国际化。

Part2. 介绍

插件国际化主要包括四方面内容： 

    1. SWT组件国际化。
    2. 程序代码国际化。
    3. 插件清单国际化。
    4. 帮助文档国际化。

下面分别介绍这几个方面的国际化方法。

Part3. SWT组件国际化

定义

SWT组件的国际化主要体现在SWT组件中的字符串的国际化，比如在确定对话框时，英文的确定按钮为OK，日文的为はい，而中文的为确定。

解决办法

生成软件产品时，采用不同语言的SWT包来打包。

Part4. 程序代码国际化

定义

程序代码国际化是指程序员编写代码中字符串（文本）的国际化，比如菜单显示内容，主界面的显示信息等。

解决办法

ResourceBundle

step1. 配置文件

创建多国语言的配置文件，文件名格式为 <resource_name>_<Language Code>_<Country Code>.properties。 例如，custom_zh_CN.properties 是简体中文的配置文件。配置文件的内容是一行一行的键值对，例如，MAIN_WINDOW_TITLE=Hello World-zh_CN是简体中文配置文件的第一条键值对。

step2. 程序代码

有了多个配置文件之后，需要设置当前采用哪个配置文件。ResourceBundle类能绑定配置文件，它可以获取指定key的value。关键代码：ResourceBundle.getBundle(BUNDLE_NAME, Locale);其中BUNDLE_NAME指定了所有配置文件共用的包路径，而Locale则指定了采用的语言种类。

step3. 使用

通过ResourceBundle.getString(key)方法，能获取配置文件中该key的value值。

package net.atp.trader.client.test.multi.way1;
import java.util.Locale;
import java.util.ResourceBundle;
public class CustomString {
    private static final String BUNDLE_NAME = "net.atp.trader.client.test.multi.way1.custom";
    private static ResourceBundle rb = null;
    public static void main(String args[]) {
        // CustomString.setLocale(Locale.getDefault());
        // CustomString.setLocale(Locale.CHINA);
        CustomString.setLocale(Locale.FRENCH);
        System.out.println(CustomString.getString(Constants.MAIN_WINDOW_TITLE));
    }
    /*
     * @param locale: 通过locale来设置初始化的资源文件
     */
    public static void setLocale(Locale locale) {
        try {
            rb = ResourceBundle.getBundle(BUNDLE_NAME, locale);
        } catch (Exception e) {
            rb = ResourceBundle.getBundle(BUNDLE_NAME, Locale.ENGLISH);
        }
    }
    /*
     * @param key: 获得key相对应的value值
     */
    public static String getString(String key) {
        try {
            String keyValue = new String(rb.getString(key).getBytes(
                    "ISO-8859-1"), "UTF-8");
            return keyValue;
        } catch (Exception e) {
            return key;
        }
    }
}

package net.atp.trader.client.test.multi.way2;
import org.eclipse.osgi.util.NLS;
public class Messages extends NLS {
    private static final String BUNDLE_NAME = "net.atp.trader.client.test.multi.way2.messages"; //$NON-NLS-1$
    public static String HelloWorld_key;
    static {
        // initialize resource bundle
        NLS.initializeMessages(BUNDLE_NAME, Messages.class);
    }
    private Messages() {
    }
}

生成的配置文件

HelloWorld_key=Hello world\!

Step4. 使用

通过Messages.key直接获取配置文件中该key的value值。

Part5. 插件清单国际化

插件清单国际化同程序代码国际化的NLS方法类似。选定plugin.xml文件，然后选择Plug-in Tools->Externalize Strings。

或者直接打开该文件，在Overview页面中选择Exporting->Externalize Strings Wizard。如下：

你也可以在该目录下创建其它语言的配置文件，如bundle_fr.properties，然后利用eclipse启动参数-nl fr来采用该配置文件。

Part6. 帮助文档国际化

在帮助系统根目录下创建nl/<language>目录，然后添加帮助文档。一般来说，帮助系统国际化工作量巨大，直接英文版即可。
